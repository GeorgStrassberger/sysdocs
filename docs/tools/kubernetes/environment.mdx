# Environment Variables

Environment variables can be defined directly within a Kubernetes **Deployment** specification:

---

Extended `httpd-deployment.yaml`:
```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: httpd-deployment       
  labels:                      
    app: httpd
spec:
  replicas: 3                 
  selector:
    matchLabels:
      app: httpd              
  template:
    metadata:
      labels:
        app: httpd            
    spec:
      containers:
      - name: httpd
        image: httpd:2.4-trixie
        ports:
        - containerPort: 80
        env:                            # ‚ö°Ô∏è NEW: Environment variables section ‚ö°Ô∏è
        - name: GEST                    # üîë KEY
          value: "Georg Strassberger"   # üí¨ VALUE
```

## Logging into a container

To connect to a running container within the cluster, use the following command:
```bash
kubectl exec -it <pod-name> -- /bin/bash
```

Before doing that, list all currently running pods to find the correct pod name:
```bash
kubectl get pods -o wide
```
Example output:
